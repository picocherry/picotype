# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

A collection of three pixel font families (`picotype`, `picotypepro`, `picosans`) with tooling to build them, patch them with Nerd Fonts icons, and publish them as the `picofonts` npm package.

## Build pipeline

The full pipeline to go from source fonts to publishable output:

1. **Build fonts with metadata** — interactive script that reads from `src/` and writes to `build/`:
   ```bash
   ./build.sh
   ```
   This runs `font-metadata-patcher.py` (requires FontForge with Python bindings: `brew install fontforge`).

2. **Patch with Nerd Fonts** — generates nerd font variants into `build/<family>nerd/`:
   ```bash
   ./generate-nerd-fonts <family>   # e.g. ./generate-nerd-fonts picosans
   ./generate                        # patches all three families at once
   ```
   Requires FontForge CLI. Uses `patcher/font-patcher` with `--simple` flag to preserve lowercase naming.

3. **Publish to npm** — copies built fonts (excluding nerd variants) to `picofonts/` and generates CSS:
   ```bash
   ./copy-to-npm.sh
   cd picofonts && npm publish
   ```

## Source structure

```
src/<family>/<style>/<family>-<style>.ttf  (or .otf)
```

Each font variant also has a `.json` (metadata) and `.png` (preview) alongside the font file. The `build/` directory mirrors this with flat layout: `build/<family>/<family>-<style>.ttf`.

Font families:
- `picotype` — 8×5px, styles: regular/bold/black (TTF)
- `picotypepro` — 10×5px, styles: regular/bold/black (OTF)
- `picosans` — 16×8px, styles: regular/bold/italic/bolditalic (TTF)

## npm package (`picofonts/`)

The npm package lives in `picofonts/` with its own `package.json`. Font files are not committed; they're copied in via `copy-to-npm.sh` before publishing.

- `picofonts/index.js` — exports CSS paths and font family metadata
- `picofonts/react-native/index.js` — exports `loadFonts`, `fonts`, `weights`, `styles`, `getFontName` for React Native / Expo
- Per-family CSS files (`picosans.css`, `picotype.css`, `picotypepro.css`) and `all.css` are generated by `copy-to-npm.sh`

## Adding small caps (OpenType smcp / c2sc)

`add-small-caps.py` adds OpenType `smcp` (lowercase → small caps) and `c2sc` (uppercase → small caps) GSUB features to built fonts. Run it **after** `build.sh` (operates on `build/`).

```bash
# Interactive — walks you through source and options
python3 add-small-caps.py

# Non-interactive
python3 add-small-caps.py --source phonetic   # default, copies glyph into phonetic slot
python3 add-small-caps.py --source capital    # lowercase → uppercase glyph when smcp applied
python3 add-small-caps.py --source lowercase  # identity substitution (placeholder)

# Single family, output to separate dir
python3 add-small-caps.py --family picosans --output build/picosans-sc
```

**Three sources** for the small-cap glyph:
- `phonetic` (default) — uses Unicode "Latin Letter Small Capital" characters (e.g. ɢ `U+0262` for G). If the phonetic slot is absent from the font, it copies from lowercase → uppercase as fallback and populates the slot so the character is also directly typeable.
- `lowercase` — uses the existing lowercase glyph (fallback: uppercase)
- `capital` — uses the existing uppercase glyph (fallback: lowercase)

Letter `x` has no standard Unicode small-cap equivalent; it falls back automatically.

## Adding old-style figures (OpenType onum)

`add-old-style-figures.py` adds the OpenType `onum` GSUB feature, mapping lining digits 0–9 to alternate figure glyphs. Run it **after** `build.sh`.

```bash
python3 add-old-style-figures.py              # interactive
python3 add-old-style-figures.py --source circled --family picosans
```

Four sources:
- `circled` (default) — ⓿①–⑨ (`U+24FF`, `U+2460–2468`); ⓿ is the filled/negative zero to match the style of ①–⑨
- `superscript` — ⁰¹²³⁴⁵⁶⁷⁸⁹ (note: ¹²³ already exist in all pico fonts)
- `subscript` — ₀₁₂₃₄₅₆₇₈₉
- `lining` — identity, placeholder

Digits whose target glyph isn't in the font yet are silently skipped — the feature wires up whatever is present.

## Direct metadata patching (without build.sh)

```bash
python3 font-metadata-patcher.py \
  --src src --family picosans --output build \
  --lowercase --type monospace --version 0.0.19
```

Key flags: `--lowercase`, `--type [serif|sans|monospace]`, `--version`, `--extension [ttf|otf]`, `--designerurl`, `--licenseurl`, `--license`.
